⚙️ Paso 0

Preparación del proyecto

Antes de escribir cualquier CSS, necesitas crear la estructura de carpetas y los archivos base. Este orden importa: cada archivo tiene una responsabilidad clara.

1
Crea la estructura de carpetas
Crea una carpeta llamada gamevault y dentro de ella la siguiente estructura. Cada archivo CSS corresponde a un nivel de Atomic Design.
gamevault/
  css/
    main.css  ← variables, reset y estilos base
    atoms.css  ← botones, badges, estrellas, portadas
    molecules.css  ← tarjeta de juego, buscador
    organisms.css  ← navbar, hero, grid de juegos
  index.html  ← la página completa
2
Escribe main.css — Variables y reset
Este archivo define las variables CSS (colores, fuentes, espaciados) y el reset base. Todos los demás archivos las usarán. Crea este archivo primero.
css/main.css
/* =============================================
   GAMEVAULT — main.css
   Variables globales, reset y estilos base
   ============================================= */

@import url('https://fonts.googleapis.com/css2?family=Exo+2:wght@400;600;700;900&family=Nunito:wght@400;500;600&display=swap');

/* ── Variables ── */
:root {
  /* Colores */
  --color-bg:        #09080f;
  --color-surface:   #13111e;
  --color-border:    #252240;
  --color-primary:   #7c3aed;
  --color-primary-h: #6d28d9;
  --color-accent:    #06b6d4;
  --color-text:      #e4e1f5;
  --color-muted:     #706c94;
  --color-star:      #f59e0b;

  /* Tipografía */
  --font-heading: 'Exo 2', sans-serif;
  --font-body:    'Nunito', sans-serif;

  /* Espaciado (usa rem para escalar con la fuente base) */
  --space-xs:  0.25rem;   /*  4px */
  --space-sm:  0.5rem;    /*  8px */
  --space-md:  1rem;      /* 16px */
  --space-lg:  1.5rem;    /* 24px */
  --space-xl:  2rem;      /* 32px */
  --space-2xl: 3rem;      /* 48px */
}

/* ── Reset ── */
*, *::before, *::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

body {
  background-color: var(--color-bg);
  color: var(--color-text);
  font-family: var(--font-body);
  line-height: 1.6;
}

img {
  display: block;
  max-width: 100%;
}

a { text-decoration: none; }
ul { list-style: none; }
¿Por qué variables CSS? Al definir --color-primary: #7c3aed en un solo lugar, puedes cambiar el color de toda la página con una sola edición. Si en dos semanas decides cambiar el morado por azul, solo modificas esa línea.
3
Crea el index.html base
Empieza con la estructura mínima. Enlaza los cuatro archivos CSS en el orden correcto — primero main.css (variables), luego los niveles de menor a mayor. Si los enlazas al revés, las variables no estarán disponibles cuando se necesiten.
index.html
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>GameVault — Tu catálogo de videojuegos</title>

  <!-- CSS en orden: base → átomos → moléculas → organismos -->
  <link rel="stylesheet" href="css/main.css">
  <link rel="stylesheet" href="css/atoms.css">
  <link rel="stylesheet" href="css/molecules.css">
  <link rel="stylesheet" href="css/organisms.css">
</head>
<body>

  <!-- Aquí irá el contenido — lo iremos agregando paso a paso -->

</body>
</html>
✦ Checkpoint — Paso 0
  • Tienes la carpeta gamevault/ con la subcarpeta css/
  • El archivo main.css existe con las variables y el reset
  • El archivo index.html enlaza los cuatro CSS
  • Abres index.html en el navegador y la página se ve negra/oscura (el reset funcionó)
⚛ Nivel 1

Átomos

Los átomos son los elementos más pequeños e indivisibles de la interfaz. No dependen de ningún otro componente. Si intentas dividirlos más, ya no tienen sentido por sí solos.

🧱
¿Qué es un átomo en GameVault?
Un botón, un badge de género, las estrellas de calificación o la imagen de portada. Cada uno existe independientemente y tiene un solo propósito. No saben nada del contexto donde van a vivir.

Crearás cuatro átomos, todos en el mismo archivo atoms.css:

A1
Átomo: Botón .btn
El botón es el átomo más reutilizable. Crearás una versión sólida (.btn--primary) y una con contorno (.btn--outline). Nota el uso de rem para el espaciado y transition para el efecto hover.
css/atoms.css
/* =============================================
   GAMEVAULT — atoms.css
   Elementos básicos e indivisibles
   ============================================= */

/* ── Átomo: Botón ── */
.btn {
  display: inline-flex;
  align-items: center;
  gap: var(--space-sm);
  padding: 0.625rem 1.5rem;
  border: 2px solid transparent;
  border-radius: 8px;
  font-family: var(--font-body);
  font-size: 0.9rem;
  font-weight: 700;
  cursor: pointer;
  transition: background-color 0.2s, transform 0.15s;
}

.btn--primary {
  background-color: var(--color-primary);
  color: #fff;
}

.btn--primary:hover {
  background-color: var(--color-primary-h);
  transform: translateY(-2px);
}

.btn--outline {
  background-color: transparent;
  border-color: var(--color-primary);
  color: var(--color-primary);
}

.btn--outline:hover {
  background-color: var(--color-primary);
  color: #fff;
}

A4
Átomo: Portada del juego .game-cover
aspect-ratio: 3/4 garantiza que todas las portadas tengan las mismas proporciones sin importar el tamaño del contenedor. object-fit: cover evita que la imagen se deforme. Agrega esto al final de atoms.css.
css/atoms.css — continúa
/* ── Átomo: Portada del juego ── */
.game-cover {
  width: 100%;
  aspect-ratio: 3 / 4;       /* Mantiene proporción de portada */
  object-fit: cover;          /* La imagen llena el espacio sin deformarse */
  border-radius: 8px 8px 0 0; /* Solo esquinas superiores redondeadas */
  display: block;
}
Previsualización

  • Ningún átomo depende de otro — cada uno funciona de forma independiente
⚗ Nivel 2

Moléculas

Las moléculas combinan dos o más átomos para formar un componente con una función clara. La tarjeta de juego no es un átomo porque depende de la portada, el badge y el rating — pero tampoco es un organismo porque es una pieza pequeña y repetible.

🧪
Regla de oro de las moléculas
Una molécula hace una cosa bien. La tarjeta de juego muestra información de un juego. La barra de búsqueda permite buscar. Si tu componente hace dos cosas distintas, probablemente deberías dividirlo.
M1
Molécula: Tarjeta de juego .game-card
La tarjeta contiene cuatro átomos: .game-cover, .game-card__title, .badge y .rating. El CSS de la molécula solo define la composición — cómo se organizan esos átomos entre sí.
css/molecules.css — nuevo archivo
/* =============================================
   GAMEVAULT — molecules.css
   Combinaciones de átomos con función clara
   ============================================= */

/* ── Molécula: Tarjeta de juego ── */
.game-card {
  background-color: var(--color-surface);
  border: 1px solid var(--color-border);
  border-radius: 12px;
  overflow: hidden;               /* Que la imagen no salga de los bordes */
  transition: transform 0.2s, box-shadow 0.2s;
  cursor: pointer;
}

.game-card:hover {
  transform: translateY(-6px);    /* Efecto de "levantar" la tarjeta */
  box-shadow: 0 16px 40px rgba(124, 58, 237, 0.25);
}

.game-card__body {
  padding: var(--space-md);
}

.game-card__title {
  font-family: var(--font-heading);
  font-size: 0.95rem;
  font-weight: 700;
  color: var(--color-text);
  margin-bottom: var(--space-sm);
  line-height: 1.3;
}

/* Flexbox para alinear badge (izquierda) y rating (derecha) */
.game-card__footer {
  display: flex;
  justify-content: space-between;
  align-items: center;
  margin-top: var(--space-sm);
}

Ahora el HTML para una tarjeta. Nota cómo los átomos del nivel anterior se usan dentro de la molécula:

🦠 Nivel 3

Organismos

Los organismos son secciones completas de la interfaz. Pueden contener moléculas, átomos, e incluso otros organismos. Ya tienen contexto: saben dónde viven en la página.

🏗️
Diferencia clave con las moléculas
El navbar sabe que está en la parte de arriba de la página. El hero sabe que es la sección principal de bienvenida. Las moléculas no saben nada de su posición — los organismos sí.
O1
Organismo: Barra de navegación .navbar
El navbar es sticky y semitransparente gracias a backdrop-filter: blur. Usa justify-content: space-between para separar logo, links y buscador. Agrega esto a organisms.css.
css/organisms.css — nuevo archivo
/* =============================================
   GAMEVAULT — organisms.css
   Secciones completas de la interfaz
   ============================================= */

/* ── Organismo: Navbar ── */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: var(--space-md) var(--space-2xl);
  background-color: rgba(9, 8, 15, 0.9);  /* Fondo semitransparente */
  backdrop-filter: blur(12px);            /* Desenfoque del fondo */
  border-bottom: 1px solid var(--color-border);
  position: sticky;
  top: 0;
  z-index: 100;
}

.navbar__logo {
  font-family: var(--font-heading);
  font-size: 1.5rem;
  font-weight: 900;
  color: var(--color-text);
  letter-spacing: -0.02em;
}

/* El "Vault" del logo en morado */
.navbar__logo span {
  color: var(--color-primary);
}

.navbar__links {
  display: flex;
  gap: var(--space-xl);
}

.navbar__links a {
  color: var(--color-muted);
  font-size: 0.9rem;
  font-weight: 600;
  transition: color 0.2s;
}

.navbar__links a:hover {
  color: var(--color-text);

O2
Organismo: Sección hero .hero
El hero ocupa el 70% del alto de la pantalla con min-height: 70vh. El gradiente radial crea un efecto de luz desde arriba. Usa clamp() para tipografía fluida que se adapta al tamaño de pantalla. Agrega esto al final de organisms.css.
css/organisms.css — continúa
/* ── Organismo: Hero ── */
.hero {
  min-height: 70vh;
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  text-align: center;
  padding: var(--space-2xl) var(--space-xl);
  /* Gradiente radial: luz morada desde arriba */
  background:
    radial-gradient(ellipse 80% 55% at 50% -5%,
      rgba(124, 58, 237, 0.35) 0%,
      transparent 70%),
    var(--color-bg);
}

.hero__eyebrow {
  font-size: 0.8rem;
  font-weight: 700;
  letter-spacing: 0.15em;
  text-transform: uppercase;
  color: var(--color-primary);
  margin-bottom: var(--space-md);
}

/* clamp(mínimo, preferido, máximo) — tipografía fluida */
.hero__title {
  font-family: var(--font-heading);
  font-size: clamp(2.2rem, 6vw, 5rem);
  font-weight: 900;
  line-height: 1.05;
  color: var(--color-text);
  margin-bottom: var(--space-lg);
}

.hero__title .highlight {
  color: var(--color-primary);
}

.hero__subtitle {
  color: var(--color-muted);
  font-size: 1.1rem;
  max-width: 500px;
  margin-bottom: var(--space-xl);

  display: flex;

📐 Nivel 4

Plantilla

La plantilla define la estructura completa de la página usando los organismos, pero con contenido de prueba o placeholders. Su propósito es validar el layout antes de insertar los datos reales.

🗺️
¿Para qué sirve una plantilla?
En proyectos reales, la plantilla es lo que presentas al cliente para aprobar la distribución de la página antes de tener contenido real. En esta práctica, te permite ver si los organismos conviven bien antes de escribir los datos de los juegos.
T1
Ensambla todos los organismos en index.html

La página es la plantilla con contenido real. Reemplazas los datos de prueba por información real de los juegos. El resultado final es lo que ve el usuario.

🏆
La diferencia entre Plantilla y Página
La plantilla define dónde va cada cosa. La página define qué va en cada lugar. En el mundo real, muchas páginas pueden compartir la misma plantilla — por ejemplo, una página de RPGs y una de Acción tienen el mismo layout pero diferente contenido.

📋 Resumen

Lo que construiste

Partiste de cero y construiste una página completa aplicando Atomic Design. Cada nivel tuvo una responsabilidad clara y ningún CSS se repitió.

gamevault/
  css/
    main.css  ← Variables CSS + reset  [base]
    atoms.css  ← .btn · .badge · .rating · .game-cover  [nivel 1]
    molecules.css  ← .game-card · .search-bar  [nivel 2]
    organisms.css  ← .navbar · .hero · .games-section · .footer  [nivel 3]
  index.html  ← Plantilla + contenido real  [niveles 4 y 5]
💡
¿Por qué esto es poderoso?
Si mañana quieres cambiar el color del botón primario, editas una línea en main.css y cambia en toda la página. Si quieres agregar un nuevo tipo de tarjeta, copies .game-card en molecules.css y la reutilizas donde quieras. Eso es la ventaja real de separar responsabilidades.
