## Escenarios de práctica para diseño de componentes de software.

**Ten en cuenta que son escenarios de práctica, no olvides estudiar otros requisitos no funcionales**

1. Sistema de reservas para una biblioteca universitaria
   - Necesitan diseñar componentes para reservar libros, gestionar préstamos y notificar vencimientos
   - Debe integrarse con el sistema de autenticación de la universidad
   - Requiere interfaz para estudiantes y otra para bibliotecarios
2. Aplicación de seguimiento de entrenamientos
   - Diseñar componentes para registrar ejercicios, almacenar progreso y visualizar estadísticas
   - Debe funcionar offline y sincronizarse cuando haya conexión
   - Considerar integración con sensores de dispositivos móviles
3. Sistema de gestión de inventario para tienda local
   - Diseñar componentes para registro de productos, control de stock y generación de alertas
   - Debe soportar múltiples terminales punto de venta
   - Requiere reportes e integración con proveedores
4. Plataforma de intercambio de apuntes universitarios
   - Diseñar componentes para subir documentos, compartir, buscar y visualizar
   - Incluir sistema de valoración y recomendaciones
   - Considerar optimización para distintos dispositivos
5. Dashboard para monitoreo de sensores IoT
   - Diseñar componentes para recibir datos de sensores, almacenarlos y visualizarlos
   - Debe procesar datos en tiempo real y mostrar alertas
   - Considerar escalabilidad y rendimiento
6. Aplicación de carpooling para la universidad
   - Diseñar componentes para registrar rutas, gestionar solicitudes y emparejar conductores/pasajeros
   - Incluir sistema de notificaciones y calificaciones
   - Considerar privacidad y seguridad
7. Sistema de gestión de turnos para clínica
   - Diseñar componentes para agendar citas, gestionar profesionales y notificar pacientes
   - Debe integrarse con expedientes médicos
   - Considerar disponibilidad 24/7 y manejo de excepciones
8. Plataforma de microblogging interna
   - Diseñar componentes para publicar, comentar y seguir a otros usuarios
   - Incluir búsqueda, notificaciones y gestión de contenido
   - Considerar escalabilidad y experiencia móvil
